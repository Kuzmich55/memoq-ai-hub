## What's New

- Replaced the first v1.0.17 build with a lighter Translation Records experience that avoids loading full history details during normal app refreshes.
- Reduced CPU and memory pressure by returning lightweight history rows in the main app state and loading full record diagnostics only when a record is opened.
- Simplified History Insights to focus on user-readable signals: average latency, slow requests, and failed requests.
- Kept issue filters and per-record diagnostics available for troubleshooting without showing provider-performance tables, cache-hit metrics, or P95-style operational details in the main view.

## Notes

- This replacement does not change the desktop version number, memoQ plugin contract, or stored history format.
- Existing history records remain compatible; full attempts, prompts, context, and segment details are still available from the record details drawer.
- Portable and update-manifest assets for v1.0.17 were regenerated from this corrected build.
