## What's New

- Added History Insights cards that summarize success rate, latency, cache hits, fallback activity, and provider/model performance for the current filtered history results.
- Added actionable insight controls so operators can jump from an attention item to matching records, filter a provider/model route, or open the related Provider configuration.
- Added History issue filtering for failed, timeout, rate limit, fallback, and slow requests, with active filter chips and one-click clearing.
- Added navigation context when opening Providers from History Insights, including a return-to-History action and focused model highlighting.
- Added per-record issue chips, diagnostic summaries, and attempt timelines so operators can see why a translation record failed or slowed down without reading raw JSON.

## Notes

- This release does not change the memoQ plugin contract or history storage format.
- Insights and diagnostics are derived locally from existing history entries, attempts, throughput, and context-source fields.
- Provider configuration actions only navigate to the relevant provider; they do not change settings automatically.
