## What's New

- Fixed memoQ plugin installation on machines where Windows marks downloaded files as blocked, preventing `MemoQ.AI.Hub.Plugin.dll` from loading with `0x80131515`.
- Translation Records now stay loaded when the app refreshes outside the History page, so records no longer flash briefly and disappear.
- Provider history metrics are preserved across non-provider refreshes and refreshed explicitly when the AI Services page is opened.
- Improved responsive layout across Dashboard, Builder, Assets, AI Services, Logs, and History for 1366px, 1024px, 768px, and phone-width windows.
- Tables now keep their information density with horizontal scrolling instead of squeezing columns into unreadable layouts.
- Drawers, modals, header controls, action bars, cards, list actions, and mobile sidebar behavior now stay within the viewport more reliably.

## Notes

- This release does not change the memoQ plugin contract, provider API contracts, stored history format, or business data structure.
- Existing translation history and provider configuration remain compatible.
- If a previously copied plugin DLL is still blocked manually, reinstalling from the desktop app or running the documented `Unblock-File` command clears the Windows file block.
