## What's New

- Fixed a Dashboard custom-install path layout issue where the Browse button could collapse into vertical letters beside the target folder field.
- Tightened input addon behavior globally so long labels and addon buttons stay readable while the input body remains shrinkable in narrow layouts.
- Improved Prompts and legacy asset-list action areas so action buttons wrap cleanly instead of squeezing nearby text.
- Scoped mobile full-width button behavior to the sticky footer and provider model manager controls instead of applying it to every responsive action bar.

## Notes

- This release is a UI-only follow-up to v1.0.18 and does not change the memoQ plugin contract, provider API contracts, stored history format, or business data structure.
- Existing v1.0.18 users will see v1.0.19 as a normal update because the desktop version and update manifest version are incremented.
