## What's New

- Added uploaded custom TM matching for `.tmx` files and existing table-based custom TM assets.
- The desktop runtime now calculates an explainable `AI Hub TM score` using token-level fuzzy matching, with memoQ-style buckets: `101%`, `100%`, `95-99`, `85-94`, `75-84`, and `<75`.
- memoQ-exported UTF-16 TMX files with `x-context-pre` and `x-context-post` metadata are now parsed, allowing exact source matches with matching neighbor context to be promoted to `101%`.
- Uploaded TM matches are passed to AI providers separately from memoQ SDK `tmHints`, so official memoQ fuzzy hints and local uploaded TM matches remain clearly distinguishable.
- Translation history now records and displays uploaded TM matches, score buckets, source/target text, and asset names for easier diagnostics.
- Repeated identical TM hits are deduplicated before the top matches are sent to AI, preventing duplicate TMX entries from filling the match list.

## Notes

- `AI Hub TM score` is an AI Hub local score inspired by OmegaT-style token matching; it is not claimed to be memoQ's internal proprietary match rate.
- Uploaded TM matches below the default `75` threshold are not sent to AI provider payloads.
- This release does not change the memoQ plugin contract or provider API configuration format.
