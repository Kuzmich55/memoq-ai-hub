## What's New

- Added the missing TM selector in Setup, so profiles can now bind uploaded Custom TM / TMX assets directly from the Assets step.
- Setup now preserves Custom TM selections as `custom_tm` profile bindings, allowing uploaded TM matches to participate in translation requests after the profile is saved.
- Setup now lets each profile choose which uploaded TM fuzzy buckets are sent to AI: `101%`, `100%`, `95-99`, `85-94`, `75-84`, and optional low-confidence `<75`.
- Prompt payloads now keep memoQ's official best fuzzy TM hint separate from uploaded TM matches. memoQ hints remain the primary project reference, while uploaded TM matches are labeled with `AI Hub TM score` as supplemental guidance.
- The Assets page now shows unbound files as `Not attached to any profile` instead of `Not available`, making it clear that uploaded files are present but not yet used by a profile.
- Large TMX files with many escaped memoQ context tags now parse correctly instead of hitting the XML entity expansion default limit.
- Custom TM parsing now keeps up to 50,000 entries for matching instead of reusing the glossary preview-oriented 1,000-row cap.

## Notes

- This release keeps the v1.0.20 uploaded TM scoring model and memoQ-style score buckets, and adds profile-level filtering for those buckets.
- Existing uploaded TM assets do not need to be re-imported, but they must be selected in Setup for the target profile before they can affect translation output.
- This release does not change the memoQ plugin contract or provider API configuration format.
