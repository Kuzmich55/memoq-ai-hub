## What's New

- Made dependency installation and Windows release packaging reproducible with a committed pnpm lockfile, frozen installs, and lockfile-based CI caching.
- Added repository governance checks to CI and limited routine CI permissions to read-only repository contents.
- Hardened the unauthenticated desktop gateway so it binds only to loopback hosts and rejects invalid ports before startup.
- Restricted external and update navigation to validated HTTPS URLs, including final redirect targets, and rejected unsafe update artifact filenames.
- Replaced a packaging test that depended on stale transitive modules with a self-contained ESM fixture.

## Notes

- Existing AI service connections, translation profiles, uploaded assets, translation records, and local storage formats are preserved.
- The memoQ plugin contract, gateway routes, Provider API configuration format, and stable update manifest schema are unchanged.
- Custom gateway hosts must now resolve to `127.0.0.1` or `localhost`; update manifests, release links, and asset downloads must use HTTPS.
- Update artifact digest or signature verification is not part of this release and remains a separately scoped hardening item.
